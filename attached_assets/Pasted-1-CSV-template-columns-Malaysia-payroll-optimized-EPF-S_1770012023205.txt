1) CSV template columns (Malaysia payroll–optimized, EPF/SOCSO optional)
✅ Recommended template (v1)

This supports HRMS-only tenants and Payroll (Malaysia). Optional statutory fields are included but can be left blank.

employee_import_template_my.csv (headers)

employeeCode,firstName,lastName,email,phone,designation,department,joinedAt,status,icNumber,taxNumber,epfNumber,socsoNumber,eisNumber,pcbCategory,bankName,bankAccountNumber,basicSalary,payFrequency

Field notes (what you should validate)

employeeCode (optional) – unique per tenant (if provided)

firstName (required)

lastName (required)

email (optional but recommended; if present validate format)

phone (optional; normalize E.164 if you already do)

designation (optional)

department (optional; can auto-create department or map existing)

joinedAt (optional) – accept YYYY-MM-DD only (avoid locale ambiguity)

status (optional) – active or inactive (default active)

icNumber (optional) – Malaysia NRIC / Passport; store as string

taxNumber (optional) – LHDN number if you store it

epfNumber / socsoNumber / eisNumber (optional) – store as string

pcbCategory (optional) – e.g., M, K, or your internal enum

bankName, bankAccountNumber (optional)

basicSalary (optional) – numeric (store as cents if you do)

payFrequency (optional) – monthly default; allow weekly, biweekly

✅ If you want a “clean HR-only template”

You can offer a second template for non-payroll tenants:

employeeCode,firstName,lastName,email,phone,designation,department,joinedAt,status

import { Tabs, TabsContent, TabsList, TabsTrigger } from "@/components/ui/tabs";

            {/* ATTENDANCE CONTENT */}
            Attendance content here…
          </div>
        ) : (
          <LockedPanel
            title="Attendance is locked"

function LockedPanel({
  title,
  message,
  onGoToAddons,
}: {
  title: string;
  message: string;
  onGoToAddons?: () => void;
}) {
  return (
    <div className="rounded-2xl border p-6">
      <div className="flex items-start gap-3">
        <div className="mt-0.5 rounded-full border p-2">
          <Lock className="h-4 w-4" />
        </div>
        <div className="flex-1">
          <div className="text-lg font-semibold">{title}</div>
          <div className="mt-1 text-sm text-muted-foreground">{message}</div>
          {onGoToAddons && (
            <div className="mt-4">
              <Button onClick={onGoToAddons}>Go to Add-ons</Button>
            </div>
          )}
        </div>
      </div>
    </div>
  );
}

B) Status filter chips (shadcn ToggleGroup)
import { ToggleGroup, ToggleGroupItem } from "@/components/ui/toggle-group";

type StatusChipsProps = {
  value: "all" | "active" | "inactive";
  onChange: (v: "all" | "active" | "inactive") => void;
  counts?: { active: number; inactive: number };
};

export function StatusChips({ value, onChange, counts }: StatusChipsProps) {
  return (
    <ToggleGroup
      type="single"
      value={value}
      onValueChange={(v) => {
        if (!v) return; // ignore empty toggle
        onChange(v as any);
      }}
      className="justify-start"
    >
      <ToggleGroupItem value="all" aria-label="All employees">
        All
      </ToggleGroupItem>

      <ToggleGroupItem value="active" aria-label="Active employees">
        Active{counts ? ` (${counts.active})` : ""}
      </ToggleGroupItem>

      <ToggleGroupItem value="inactive" aria-label="Inactive employees">
        Inactive{counts ? ` (${counts.inactive})` : ""}
      </ToggleGroupItem>
    </ToggleGroup>
  );
}
