You are building a multi-tenant SaaS platform (MyBizStream). Implement a complete **Tenant Sign-Up Flow** with subscription selection and dashboard feature gating. Ensure the following:

1️⃣ **Sign-Up Page**
- Web and Flutter mobile pages
- Collect tenant info: 
    - Tenant Name, Business Type, Subdomain, Country
    - Admin User: Name, Email, Password
    - Optional fields: Logo, Contact Number
- Validate inputs (Zod for backend, React Hook Form for frontend)
- On submission, create tenant in `tenants` table and admin user in `users` table
- Generate default `tenantId` and assign to the admin user
- Return tenant context + temporary JWT

2️⃣ **Subscription Selection Page**
- After sign-up, redirect tenant to subscription page
- Show available subscription tiers (Free, Starter, Pro, Enterprise)
- Display modules included per tier with icons or checkmarks
- Let tenant select desired bundle
- Store tenant’s selection in `tenantSubscriptions` table
- Calculate country-specific pricing and taxes using `countryPricingConfigs` and `exchangeRates`
- Trigger `baseNotificationService` to notify platform owners of new subscription

3️⃣ **Dashboard Routing**
- After subscription selection, redirect tenant/admin to dashboard
- Dashboard should dynamically **enable/disable modules** based on selected subscription
- Modules not included in subscription must be hidden or show "Upgrade" CTA
- Ensure **RBAC + feature-gating middleware** checks tenant subscription for all API requests

4️⃣ **API Implementation**
- `/api/auth/signup` → create tenant + admin, return JWT
- `/api/subscription/select` → store tenant subscription
- `/api/dashboard` → dynamically fetch enabled modules based on tenant subscription
- Middleware:
    - `authenticateJWT`
    - `tenantResolutionMiddleware`
    - `subscriptionCheckMiddleware` (verify selected modules)

5️⃣ **Mobile (Flutter)**
- Sign-up page with tenant + admin fields
- Subscription selection page
- Dashboard navigation reflects subscription features
- Bloc pattern: SignUpBloc → SubscriptionBloc → DashboardBloc
- Offline support: store selected modules locally until server confirms

6️⃣ **Success Metrics**
- Tenant can sign up → select subscription → access dashboard without error
- Dashboard only shows modules enabled in subscription
- Notifications sent to platform owners
- Country-specific pricing applied correctly

7️⃣ **Bonus**
- Add smooth onboarding animation/progress bar: SignUp → Subscription → Dashboard
- Optional: pre-fill tenant country based on geolocation
