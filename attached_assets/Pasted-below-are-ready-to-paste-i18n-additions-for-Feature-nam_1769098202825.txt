How to choose tone

Default: friendly for India SMB

Default: formal for Super Admin / enterprise screens

In code:

const tone = isAdminArea ? "formal" : "friendly";
t(`lockedFeature.tones.${tone}.planLocked.title`, { feature });


(You can also store tone in tenant branding settings later.)

3) A/B copy variants for conversion testing

Add under lockedFeature.ab.A and lockedFeature.ab.B.

A = benefit-first (works better in India)

B = limit/urgency-first (works when user hits a limit)

EN (en.json)
{
  "lockedFeature": {
    "ab": {
      "A": {
        "planLocked": {
          "title": "Unlock {{feature}} with an upgrade",
          "description": "Get more users, higher limits, and access to {{feature}}.",
          "cta": "Upgrade"
        },
        "addonLocked": {
          "title": "Enable {{feature}} instantly",
          "description": "Start your {{days}}-day free trial and activate in seconds.",
          "cta": "Start trial"
        }
      },
      "B": {
        "planLocked": {
          "title": "{{feature}} is locked on your current plan",
          "description": "Upgrade now to continue without interruptions.",
          "cta": "Upgrade now"
        },
        "addonLocked": {
          "title": "{{feature}} is not enabled",
          "description": "Install this add-on to proceed. Trial: {{days}} days.",
          "cta": "Enable"
        }
      }
    }
  }
}

How to pick A/B variant (simple, stable)

Assign per tenant once, store in localStorage or tenant setting:

app:ab:lockedFeature = "A" | "B"

In code:

const ab = getOrCreateAbVariant("lockedFeature"); // returns "A" or "B"
t(`lockedFeature.ab.${ab}.planLocked.title`, { feature });


✅ Best practice:

Use A by default

Switch to B only for “limit reached” moments

Implementation Notes (quick + practical)
Recommended resolution order in code

Feature name: features.*

Copy layer:

If A/B enabled → use lockedFeature.ab.*

Else use lockedFeature.tones.*

Else fallback to base lockedFeature.*

Example helper
function lockedCopyKey({ tone, ab, reason, field }) {
  if (ab) return `lockedFeature.ab.${ab}.${reason}.${field}`;
  return `lockedFeature.tones.${tone}.${reason}.${field}`;
}
