Add PAYROLL as a new feature module (plan-gated) integrated with existing HRMS.

Context:
We already have HRMS (employees, attendance, leaves, timesheets) and a feature flag system with plan builder.
Add payroll as a module (not a business type), India-first, v1 = payroll ops (not statutory filing).

1) Feature Catalog + Plan Builder
- Add feature flags to FEATURE_CATALOG:
  - payroll
  - payroll_payslips
  - payroll_payout_export
  - payroll_compliance_tracking
- Add to plan builder UI grouped under "HRMS / Payroll"
- Default India plans:
  - Free: payroll false
  - Basic: payroll true but limited by employee limit (see limits)
  - Pro: payroll true full

2) Limits
- Add limits to LIMIT_CATALOG:
  - payroll_employees (int, -1 unlimited)
  - pay_runs_per_month (int, -1 unlimited)
Default:
  - Basic: payroll_employees=10, pay_runs_per_month=1
  - Pro: payroll_employees=-1, pay_runs_per_month=-1

3) Database Schema (PostgreSQL + Drizzle)
Create tables with tenant_id scoping + audit logging:
- payroll_settings:
  tenant_id PK/FK, currency_code, pay_cycle_day, pf_enabled, esi_enabled, pt_enabled, created_at, updated_at
- salary_structures:
  id, tenant_id, employee_id, effective_from, basic, hra, allowances_json, deductions_json, ctc, is_active, created_at
- pay_runs:
  id, tenant_id, month, year, status(draft/approved/paid/cancelled),
  generated_at, approved_at, paid_at, created_by, approved_by
- pay_run_items:
  id, tenant_id, pay_run_id, employee_id,
  earnings_json, deductions_json, gross, total_deductions, net,
  attendance_days, unpaid_leave_days, overtime_hours, notes
- payslips:
  id, tenant_id, pay_run_item_id, pdf_url, issued_at

4) Payroll Engine (server/services/payroll)
- generatePayRun(month, year):
  - pulls active employees within payroll_employees limit
  - reads salary_structures effective for the month
  - reads attendance/leaves summary for the month
  - computes:
    gross, deductions, net (simple rules first)
  - creates pay_run + pay_run_items
- approvePayRun(payRunId)
- markPaid(payRunId) (manual for now)

5) API Endpoints (RBAC + feature gating)
All endpoints must:
- requireAuth (session or bearer)
- requireFeature("payroll")
- tenant isolation
Routes:
GET /api/payroll/settings
PATCH /api/payroll/settings
GET /api/payroll/salary-structures
POST /api/payroll/salary-structures
PATCH /api/payroll/salary-structures/:id
GET /api/payroll/pay-runs?page=&limit=
POST /api/payroll/pay-runs/generate { month, year }
POST /api/payroll/pay-runs/:id/approve
POST /api/payroll/pay-runs/:id/mark-paid
GET /api/payroll/pay-runs/:id/items
GET /api/payroll/payslips/:itemId/pdf (generate on demand)

6) Frontend UI (shadcn)
Add sidebar section "Payroll" (only if feature enabled):
- Payroll Dashboard (summary widgets)
- Salary Structures
- Pay Runs
Pay Runs page:
- Generate Pay Run (month/year)
- Table with status, totals, actions (approve/mark paid)
Pay Run details:
- Employee list, net pay, search/filter, download payslip

7) PDF Payslips
- Reuse existing PDF infrastructure (invoice-pdf patterns)
- Add payslip template with tenant branding

8) Tests
- Feature gating: payroll disabled -> 403 MODULE_NOT_AVAILABLE
- Generate pay run creates items correctly
- Limit enforcement: payroll_employees respected
- Tenant isolation tests
- UI: payroll menu hidden when not enabled

Acceptance:
- Payroll module appears only when enabled
- Can generate a pay run and see per-employee net pay
- Can approve and mark paid
- Can download payslip PDF
- No regressions in existing 116/116 tests (or current suite)
